Serving

Serving이란 ML 모델 사용을 위한 API를 제공하는 것입니다.

이를 위한 다양한 방법들이 존재합니다.

너무 많은데 일단 제가 아는 선에서 정리하고 차차 글을 발전시켜 나갈 생각입니다.

Model에 데이터를 넣고 결과를 받으려면 학습에 사용했던 이미지 형식과 동일하게 하기 위한 preprocessing 작업이 필요합니다. 학습용 코드를 추가합니다.

1# train.py
2import tensorflow as tf

4

7train_x, test_x = train_x / 255.0, test_x / 255.0
8

9model = tf.keras.Sequential([
10    tf.keras.layers.Flatten(input_shape=(28, 28)),

15model.compile(

18    metrics=['acc']

20

21model.fit(train_x, train_y, epochs=5)
22loss, acc = model.evaluate(test_x, test_y)
23print("Model Loss : {:.4f} Model Acc : {:.4f}".format(loss, acc))
24

25model.save("models/mnist_dnn.ckpt")

코드를 확인해보면 input_shape 으로 (28, 28) 인 것을 확인할 수 있습니다. 따라서 앞으로 만들 Serving 파일에도 이를 위한 과정들이 있을 겁니다.

Flask에서 라우터 생성은 파이썬 함수에 @app.route라는 데코레이터를 통해 만들 수 있습니다. Router는 /prediction 을 통해 접근할 수 있고 접근 방식은 POST로 설정해줬어요. 이미지를 입력받기 위함이겠죠?

3-1 이미지 입력받기

1image = request.files.get("image")
2pil_image = Image.open(image)

Flask에서 파일이 포함된 request를 받으면 request.files에 ImmutableMultiDict 형태로 저장이 되는데 이를 get 메소드를 통해 읽은것이다. image라는 key 값은 내가 그렇게 지정해서 파일을 보낼 것이기 때문에 그렇게 읽었다. 그렇게 image라는 변수에는 FileStorage라는 클래스로다음과 같은 파일이 들어있는데 이는Image.open` 을 통해 PIL Image로 변환이 가능합니다.

3-2 이미지 변환하기

1pil_image = pil_image.resize((input_shape[1], input_shape[2]))
2pil_image = pil_image.convert("L")
3

4numpy_array = np.array(pil_image)
5numpy_array = numpy_array / 255.0
6if numpy_array.sum() > 200:
7    numpy_array = 1 - numpy_array
8prediction_array = np.array([numpy_array])  # (28, 28) to (1, 28, 28)

모든 사용자가 제가 이미지를 입력한대로 (28, 28)에 grayscale 이미지를 입력한다면 참 좋을텐데 그렇게 강제할 순 없겠죠?

  • resize 사용자의 입력 이미지가 (28, 28) 일수도 있지만 (29, 29)일수도 있고 (2000, 2000)일수도 있어요.
  • convert 이미지에는 "모드"라는 개념이 있어요. RGB, L, P 등이있는데 L은 grayscale을 의미해요. 제가 학습시 사용했던 데이터도 grayscale의 이미지였답니다.
  • np.array 이미지에 대한 근본적인? 처리는 pillow 이미지 형태로 하는 게 편하지만 데이터의 측면에서는 numpy의 형태로 변환시킨 이후 진행하는 게 편하답니다.
  • numpy_array / 255.0 학습때도 했던 것 처럼 정규화기법이에요.
  • numpy_array.sum() > 200 학습때 사용했던 데이터처럼 숫자가 흰색이고 배경이 검정인 이미지를 넣으면 잘 분석하는데 숫자가 검정이고 배경이 흰색인 이미지를 넣으니 당연히 제대로 분석을 못하더라구요. (28, 28)의 이미지를 255.0으로 나눠준 후 값을 전부 더하다보니 배경이 흰색이면 200이 넘고 배경이 검정이면 200을 넘지 못하더라구요.
  • 1 - numpy_array 배경이 흰색이면 색상을 반전시켜줬어요. 배경이 검정이고 글이 흰색이 되겠죠?
  • np.array([numpy_array]) 모델에 대한 입력을 기본적으로 배치를 포함해서 받아요. 저는 1개의 이미지에 대한 결과값을 반환할 것이니 단순히 차원을 하나 추가해줬어요. (28, 28) -> (1, 28, 28)

3-3 모델에 입력

1predictions = model.predict(prediction_array)
2prediction = np.argmax(predictions[0])

드디어 모델에 입력을 주고 결과를 받네요. 배치를 포함한 입력을 줬으니 결과도 배치를 포함한 결과를 받을 거에요. 그래서 [0]로 첫번째 인덱스 결과를 받아와야 한답니다. 0~9까지의 결과값에 대한 확률로 데이터가 나오는데 그 중 가장 큰 값이 제가 원하는 결과값입니다.

3-4 결과 반환

1return {"result": int(prediction)}

전체 소스코드

1from flask import Flask, request
2from PIL import Image
3import numpy as np
4from tensorflow.keras.models import load_model
5

6model_path = "models/mnist_dnn.ckpt"
7model = load_model(model_path)
8

9input_shape = model.layers[0].input_shape
10

11app = Flask(__name__)
12

13

14@app.route("/")
15def root_route():
16    return {"error": "use POST /prediction instead of root route"}
17

18

19@app.route("/prediction", methods=["POST"])
20def prediction_route():
21    image = request.files.get("image")
22    pil_image = Image.open(image)
23    pil_image = pil_image.resize((input_shape[1], input_shape[2]))
24    pil_image = pil_image.convert("L")
25

26    numpy_array = np.array(pil_image)
27    numpy_array = numpy_array / 255.0
28    if numpy_array.sum() > 200:
29        numpy_array = 1 - numpy_array
30    prediction_array = np.array([numpy_array])  # (28, 28) to (1, 28, 28)
31    predictions = model.predict(prediction_array)
32    prediction = np.argmax(predictions[0])
33    return {"result": int(prediction)}
34

35

36if __name__ == '__main__':
37    app.run(debug=True)
